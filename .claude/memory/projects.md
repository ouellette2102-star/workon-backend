# Active Projects

## WorkOn — Global Work Infrastructure

### Vision
Universal work layer connecting demand signals with human capability through contracts, availability, and reputation. Uber + LinkedIn + Shopify + TaskRabbit combined. Building the global work network.

### Current Phase: Quebec MVP
Proving the model in a single market before scaling.

### Stack
- **Backend**: NestJS + Prisma + PostgreSQL (this repo)
- **Frontend**: Next.js + Clerk Auth
- **Mobile**: FlutterFlow
- **Deploy**: Railway (production)
- **Payments**: Stripe Connect (escrow model, 15% commission)
- **Auth**: JWT (native) + Clerk (legacy)

### Platform Components
1. Demand signal matching (mission posting + discovery)
2. Capability graph (worker profiles, skills, ratings)
3. Instant contracting (digital contracts, e-signatures)
4. Availability signaling (location + schedule)
5. Portable reputation (reviews, trust tiers)
6. Secure payments (Stripe escrow, instant worker payouts)

### Key Modules (24 active)
Auth, Missions, Offers, Payments, Chat, Contracts, Reviews, Notifications, Workers, Map, Compliance, Audit, Security, Scheduling, Support

### Business Model
- 15% commission on completed missions (Stripe escrow)
- Future: API access, premium features, enterprise tier, white-label

### Market Expansion Path
Quebec → Ontario → BC → Canada → North America → Global

### Current Risks
- Supply/demand chicken-and-egg in early markets
- Dual auth system (JWT + Clerk) adds technical debt
- FlutterFlow may limit mobile capabilities at scale
- Quebec-only limits growth velocity short-term

### Strategic Priorities
1. Prove unit economics in Quebec (commission per transaction)
2. Build supply density in Montreal (workers)
3. Activate demand channels (SEO, Facebook, referral)
4. Validate repeat usage (client comes back)
5. Prepare for geographic expansion
